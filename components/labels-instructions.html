Labels & Instructions

Labels

Everything a user can interact with needs a label. These labels need to be programmatic, meaning a computer program should be able to tell which labels are for which component.

Labels should be persistent and visible.

On the web, there are a couple different ways to attach a label programmatically. For instance, with an HTML button, the text within the button element is the label. For inputs, you'll usually use a label element. Feildsets also provide labels. You can also add or extend labels with the aria attributes.

The aria-label and aria-labelledby attributes are useful, for extending the uniqueness of the label. For any component you build start with the HTML recommendation for associating a label.

An aria-labelledby attribute points to one or more ids to create an accessible label. An aria-label is defined by a text like "button label."

Visible & Accessible Labels

Visible labels are any visible text labelling a user interface components. Always present it as text rather than text in an image. The accessible label is the label that is programmatically available to other technologies, including assistive technology. Sometimes the visible and accessible label are the same. Sometimes the accessible label has additional text.

Labels should be visible in text. The one exception may be a search bar where the "Search" button is represented with the icon of a magnifying glass. Users are familiar with this convention. Remember, even if you use an icon visually, you have to include an accessible text label.

The visible label must match the accessible label. What does that mean? It means any visible text labelling an ui component needs to match or start the accessible label.

Unique Labels

Unique labels are important. We need to know what each UI component is and if there are multiples of the same type we need to know how they are different from one another.

Let's take as an example a library catalog search page with multiple books listed. The catalog entries are the title, some metadata, and then a button to reserve the book. Each button has label "Reserve" programmatically and visually. The reserve button is visually close to the title it reserves for context. However, when I'm skimming through these buttons with a screen reader, I just hear "Reserve...Reserve...Reserve." It would be more useful if the buttons included the title of the book.

The two things I want to name the button are already on the page, the label "Reserve" and the title of the book. So, I give each of those things a unique id and reference them with an aria-labelledy.

Now when I skim the buttons my screenreader will read "Reserve The Very Hungry Caterpillar". When looking at the page with a screen magnifier, the buttons may still be somewhat similar so make sure to use grouping and other visual cludes to connect the button and the title.
