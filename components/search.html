Search Region

Add a role="search" to the section of the page with search. This should include elements like the search input and button, as well as any filters, and the search result sets.

Using this attribute will create a search landmark role. Attach an aria-label or aria-labelledby to create a more descriptive region name or if there are multiple search regions on the page. (For example, if you have a site search and a catalog search.)

Search Filters

Search filters can apply automatically or they can wait to be applied til the user selects an apply button. There are pros and cons to both options.

Researchers often want to apply a number of filters at once.

Single Page Applications versus New Page Load

Single page applications (SPA) describes a way of working with websites and data. With single page applications, only the information changes. You can have a single page application with dynamic or applied filters.

Updating the User on Search Results

People need to know that search results have changed. If your search results cause a page reload, the user will have the standard information available when navigating to a new page.

In other cases, you'll want to use an aria-live region. Make it polite; make sure it's always there. Typically you'd use something like the "N results found".

Managing Focus

Pay attention to your focus management. When a user applies filters, keep the focus in the same place. Avoid changing where focus is; it is disorienting to the user. In some cases, where the filter options change, you may have to move the focus because the list changes. If that's the case, keep focus as close as possible.

Search Organization

Organize search results in a list. Generally, each item in a search result would have a heading level.

Persistence of Search Terms

Try to incorporate the search term in the page. Sometimes you'll find a search interface where the search term is only available in the search bar, similar to placeholder text. It's better to make it a fixed part of the page. One pattern is to include it in the total search results messaging eg "45 results found for "Seaweed"".

Error messages

Put some thought into your "no results found" message. If you can, add in functionality like search suggestion, ie "did you mean to search for", to help with spelling errors. Give users tips on how get search results, for example "try removing some filters".
