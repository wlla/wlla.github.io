Standard makers

World Wide Web Consortium

The World Wide Web Consortium (W3C) has been creating and maintaining web standards since the web was born! It's an international body creating a number of standards relevant to accessibility including the main standards explicitly on accessibility. There are a number of working groups in the W3C creating and maintianing these standards. (And many ways to get involved.) The Web Accessibility Initiative works directly on accessibility guidelines.

Web Content Accessibility Guidelines (WCAG)

Creator:
W3C
Standard:
Web Content Accessibility Guidelines (WCAG) 2.1

The Web Content Accessibility Guidelines are the international standard for digital accessibility. Don't let the "Web" in the title throw you. These standards are technology agnostic; meaning they definitely apply to web technologies, but the principles and success criteria can also be applied to the technology stacks of native mobile apps, virtual reality experiences, et al.

Officially adopted Summer 2018, WCAG 2.1 is the most up-to-date standard.

Authoring Tool Accessibility Guidelines (ATAG)

This standard helps us build authoring tools that are accessible (everyone can use the interface) and that help create accessible content. So, the standard has two parts. The first is that the interface follow WCAG. The second section are guidelines for how an authoring tool can make sure the resulting content is accessible.

Interfaces that should follow this guideline are those used to create digital material. Examples would be, the comment section of website, a Twitter client, a WYSIWYG (What You See Is What You Get) editor in a CMS (Content Management System.)

Accessible Rich Internet Applications (ARIA)

When people use technologies like JavaScript or write their own custom widgets, information on these changes and UI components aren't necessarily surfaced for assistive technology. The ARIA standard gives people a way to define these changes and components in a way expected by screenreaders. As much as possible, we should try to build things that already surface this information expected ways. This is why the best ARIA is no ARIA, meaning a product was built in a way that doesn't require the addition of ARIA or where ARIA is additive. Sometimes, updates in other standards will make ARIA defunct. For instance, HTML5 includes elements that take care of some of the work of the ARIA role attribute.

The ARIA properties are implemented as HTML attributes. The Labels & Instructions page covers examples of ARIA attributes being used to label UI elements.
