Images

All meaningful images need text providing equivalent description of the content.

Simple Images

In most digital technologies, the text alternatives for simple images are provided with alt text. In HTML, alt text for an img is included with the alt attribute. Alt text isn't exhaustive, it should be convey what the image is meant to convey. To help get to that, ask yourself: What, When, Where, Why (am I including this image)

Some additional guidance on alt text:

  • Don't include quotation marks in alt text. (They can cause a screenreader to truncate the text.)
  • Only identify the material type if it is significant. The screenreader will announce that it is an image so starting alt text with "Image of..." is repetative. You may want to identify the format if it's meaningful. If for instance it is important that the image is a photo or etching or watercolor, then include that information.
  • Be concise: Think about writing alt text that could fit in a tweet with room to spare. Screenreader users want to get the information and move on. Also, alt text is read out all in one go without the finer control you find elsewhere to read just sections of the text.

If an image is linked, the alt text should be meaningful link text. Imagine an company, Big Corp, that has social media icons in their footer. The icons link to the respective Big Corp social media profile. If you have an Instagram logo, the alt text "Camera" isn't useful. Instead, use alt text like "Big Corp on Twitter."

Some additional reasons you need alt text: Alt text is what displays if an image doesn't load. There are a couple reasons an image won't load: poor internet connection, user preferences, and security settings. Alt text are also scooped up as fuel for search engines, improving SEO.

When creating content, authors will often have the option to add a title and/or description field. It's the description field that pretty much always maps to and creates the alt text. (The title usually defines the text available by hovering over an image.)

Simple Images & Screenreaders

When you hit an image with a screenreader, it will announce the format (image) and the alt text, if there is any. If you haven't included an alt attribute, it will simply announce that it is in image and the user will have to wonder what it is. If you haven't included alt text and it's a link, the link url will be read out. If the image is decorative so you've included an empty alt attribute, the screenreader will ignore the image completely and skip right over it.

Complex Images, Graphics, and Charts

When you're working with complex images, graphics, and charts, standard alt text might not be enough. Remember, alt text should be short. When this is the case, you'll want to create alt text and a longer description. The longer description could take a couple different forms and be presented in a couple different ways. You could include it on another web page or as a download. The best bet would be to include it on the same page as the complex media. If you're presenting a bar chart of yearly sales, include a data table of the source data. The alt text should summarize what you're trying to convey and reference the longer description, for example "Yearly sales increased from 2014-2017, but started to drop off since. Data table in Appendix A."

With the longdesc attribute, you can link to the longer description. This attribute is only to link to that information, don't try to include the text of the longer description in the attribute.

The DIAGRAM center has resources on how to write good image description, including for complex images. They also have a POET tool to help learn good image description. DIAGRAM Resources on complex image accessibility.
