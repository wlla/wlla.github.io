Media Players

Media players must have accessible interfaces and follow WCAG. Players also need to support the range of formats of text alternatives we use for time-based media. These formats include captions, transcripts, text video description, and audio description.

AblePlayer also support rendering video description as text that will be read aloud by a screenreader.

You can configure how captions appear; changing things like position on screen, color, and background color. It's great if a player supports interface controls for a user to customize settings to their own preference.

If a player doesn't support toggling on and off audio descriptions, upload both as seperate videos and make sure they are shared on the same page or they link to each other. This is also a good approach when your video has extended audio description where the described video is longer than the undescribed.

Players

AblePlayer

AblePlayer Website

AblePlayer is free and open-source. It is highly customizable and supports audio description, captioning, interactive transcripts. It's also got a feature where you can feed it video description as time-stamped WebVTT tracks that will appear in aria-live regions; meaning the text will be spoken by a screen-reader. Note: this approach requires Javascript. This would be an alternative to creating a narration track.

OzPlayer

OzPlayer bills iteself as the world's first completely accessible video player. It's free for non-profits with a budget under a million. It doesn't currently have an audio counterpart.

OzPlayer Website

YouTube

It's not perfect, but YouTube has some okay accessibility.

Note: If you're embedding content from Youtube, make sure that your embedded vido are pulling in the captioning of the original track. It takes an extra flag in the embed code. (If you don't take this step the embedded video will be missing the captions, yikes!)
