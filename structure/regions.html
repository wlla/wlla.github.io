Consider this webpage. Think about when you first visited. You probably scaned it and got a sense of the different high level sections of the page: the header, navigation sections, and the main section. Depending on where you came from you then focused on one of those areas to complete your task be it read about regions or navigate to another section.

In HTML, these high level chunks of the page are landmark regions. You scan them with a screenreader or visually. To make sure that they are available to assistive technologies, you need to make sure that you are semantically marking up your landmark regions.

Landmark regions may also be referred to as navigational landmark regions or navigational landmarks.

Landmark Regions

You can create semantic landmark regions with HTML5 elements and ARIA landmark roles. You should only use the ARIA roles if there is no equivalent landmark region in HTML5.

Using certain HTML5 sectioning elements, like header or main, will automatically create a landmark region. With ARIA you add landmark role attributes to an element to create a landmark region. Add the role attribute and define it with one of the eight ARIA roles: banner, navigation, main, complimentary, contentinfo, search, form, application. Application role is tricky and changes keyboard behavior.

Landmark Region Checklist

The rest of this page has more details on regions and how to create the, but for a quick checklist review the following.

  • Make sure all the content of a webpage is within a region.
  • Some types of landmark regions, you should only use once at the top level, some can be used multiple times. These are noted in the table below.
  • Similar to headings, think about what is a manageable and useful number of landmark regions. A simple page probably won't have 20 regions.
  • Every webpage should have a `main` and one `main` only.
  • The main region is often the target of a skip link.
  • You can add names to regions using aria-label or aria-labelledby.
  • When you have multiple regions of the same type, they must have unique and descriptive names.

Naming Landmark Regions

When you have multiple regions of the same type or when you are using a generic region type like section create an accessible name. Add a unique and descriptive label with an aria-label or aria-labelledby attribute.

For example, this page has a navigation region with links to all the pages in the Structure section of the site. I've added an aria-label="Secondary". to make it more descriptive. A screenreader announces the name and the type of region so it is identified as "Secondary navigation region".

Screenreaders & Regions

With screenreaders, you can skim through all the regions in a page in order to navigate to important sections and get a sense of the page content (the way some users might visually scan for that info.) When navigation through the content of a page, a screenreader will also announce when you enter or leave a region.

A screenreader will announce the region type and, if there is one, region label.

There are keyboard shortcuts for skimming through all the regions in a page or through all the regions of a certain type in a page.

When you first navigate to a page with a screenreader, it may announce the total number of regions on a page.

Assistive Technologies & Regions

Assistive technologies can also mask certain landmark regions. Reader views might show a view of just the main element of the page. Reader views are often built into browsers on mobile, but a user could have similar settings for any device. These viewing options can improve legibility and remove distractions.

Checking for Regions

On webpages, you can check for headings with the WAVE tool or the aXe tools. You can also check with the visual bookmarklet. You can also check with a screenreader by skimming through regions
