Titles & Language

A title is a specific metadata field with an understandable title for the digital webpage, document, or material. The Language is a metadata field programmatically marking the language of the page or content. Both these fields are required. They are metadata of the material.

Title

In a webpage, the title is the text that appears in a browser tab. If they have a lot of tabs open, the title is what people skim to know which tab is which. The title of page is set by the HTML title element in the head.

Frequently, a tab will also include a "favicon". A "favicon" is a little icon, usually a logo, that people add to their sites. In Twitter, the Favicon is their blue bird logo, in the USA Govenment Jobs site, it is an icon of the USA Flag.

Titles should be unique and descriptive. In a website, having all the pages titled the same way is not helpful.

In most cases, the title should be the same as the Heading One (h1) of the page. On the web, you may want to identify the site itself; this should follow the title for the main page. For example, on Wikipedia the titles follow the convention of "Name of article" + "- Wikipedia". While the h1 of the page is just the "Name of article". So the article on Universal Design has the title "Universal Design - Wikipedia".

In a digital document, the title is the human readable title; it is what is announced when a screenreader opens a file. If a title is missing, the screenreader will simply announce the file name, which can be less than meaningful.

Language

Language programmatically marks what language the material is written in. This needs to be explicitly set so screenreaders can know what language to read in. If it is missing, the screenreader will need to make a guess. If I'm using a screenreader where the default is English language and it tries to read Spanish language text, the pronunciation will be completely incorrect. If it tries to read text in a non-Roman alphabet, it won't announce anything.

On the web, you set the language with the HTML lang attribute. Make sure you use a valid language code. Language codes can be specific, for instance, there is a code for Castilian Spanish.

In XML, you set the language with the xml:lang attribute. In documents like Word and PDF, you can set the language in the documents metadata settings.

If you have a subsection of your material that is written in different language, you'll want to tag that section with the appropriate language. In HTML, you still use the lang attribute, you just add it to the element or wrap the text in a span and add it there. Note: you don't need to do this for words and phrases that are proper nouns, technical terms, or that have become part borrowed words in the main language of the material.

Checking for Titles and Language

You can use automated tools to check for the prescene or absence of these tags, but you will always need to do a manual check to make sure the fields are meaningful and accurate.

On a webpage, you have a couple of options:

  • The WAVE tool or the aXe tools
  • Skimming through the headings with a screenreader
  • Checking the code of your page and searching for the "title" and "lang".

For different authoring software and documents like PDFs, EPUBs, and MS Word docs, you can check the documents metadata. There are also built in accessibility wizards and checkers that will alert you if a metadata field is missing.
