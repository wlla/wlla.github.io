Lists and Tables

Lists and tables organize information in specific ways. They need to be marked up semantically. On the web this means using appropriate HTML; in document formats, this means using semantic styles and mark-up.

Lists are any grouped set of things. Tables are a very specific format that is only ever be used for tabular data.

Lists

There are three types of lists: ordered, unordered, and definition lists

Feel free to nest lists, but make sure to do it semantically.

On the web, you can title a list with the aria-label or aria-labelledby attributes.

With CSS, you can customize how lists display, but the principles of being a set of things remains. On this web page, the header navigation links and the secondary navigation links are both organized in lists, but appear differently in the visual styling. For both, each link is a list item. In the header all the list items appear in the same horizontal line In the second ary navigation, the list items are organized in a column.

While the visual design is different it conveys the same message as as the semantic structure: these are a set of things. What you don't want to do is use CSS to make something *look* like a list without making it a semantic list.

Unordered & Ordered Lists

Unordered lists might be the most common type of lists. The header and secondary navigation sections of this webpage are organized as unordered lists.

Ordered lists are sequential. List items are organized by orders of numbers, letters, or Roman numberals.

Definition Lists

A definition list is a slightly more complex. This is a list where each item is made up of a key pair: a term and that term's definition.

In a library catalog, you might use a definition list in a catalog record when dealing with metadata. The metadata field label would be the term and the entry would be the definition.

I have organized the Words and Phrases page as a definition list.

Screenreaders & Lists

When you hit a list with a screenreader, the screenreader will that it is a list and the total number of list items. There are keyboard shortcuts for navigating through items on a list and for navigating to all the lists on a page. As you move through the items in a list, the screenreader may also tell you the order of that item in the list.

Tables

Never use a table to organize things visually!! Only use tables to organize and present tabular data.

A table has headers and data cells. Be sure to mark them up accordingly. On the web, always use a scope attribute for headers. (This attribute tells assistive technology if the header labels a row or a column.) With tables, you can add a caption describing the table content and summary explaining the table structure.

If you find you're making a table with multiple rows of headers, take a step back. You probably should be making multiple tables each with fewer headings.

Screenreader & Tables

When you hit a table with a screenreader, the screenreader will announce that it is a table. Depending on your screenreader settings, each cell may also be announced with its associated header. (As long as you have correctly marked up the table and used the scope attribute.)
